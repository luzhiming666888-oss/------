**任务**：填写 `src/avl.c` 中所有 TODO 函数，同步更新 `report_draft.md`。

**实现内容**（11 个函数）：
- `avl_new_node` — 内部辅助，创建新叶节点
- `avl_destroy` — 后序遍历销毁，free(data) + free(node)
- `avl_rotate_ll` — LL 型右旋
- `avl_rotate_rr` — RR 型左旋
- `avl_rotate_lr` — LR 型双旋（先左旋左子，再右旋 z）

- `avl_rebalance` — 内部平衡修复，根据 BF 判断四种失衡类型
- `avl_insert` — 递归 BST 插入 + 回溯 rebalance
- `avl_delete` — 递归 BST 删除（前驱替换策略）+ 回溯 rebalance
- `avl_search` — 迭代 BST 查找 O(log n)
- `avl_inorder` — 递归中序遍历填充数组

**删除策略细节**：前驱替换时，将前驱的 data 指针转移给目标节点后，前驱 data 置 NULL，防止递归删除前驱时双重释放。C 标准保证 free(NULL) 安全。

**关键 Bug 与修复**：
1. **scanf/fgets 混用导致缓冲区错位**：scanf("%d") 读取数字后留下 \n，fgets 读到空行 → 全部统一用 fgets + atoi/atof
2. **fgets buf_size 过小导致行截断**：STUDENT_ID_LEN=13 传给 fgets 时只能读 12 字符，学号正好 12 位，\n 残留在 stdin → read_line 中检查 fgets 是否读到 \n，若未读到则消耗剩余字符
3. **EOF 无限循环**：管道输入耗尽后 scanf 返回 EOF 导致 while(1) 死循环 → 检测 feof(stdin) 优雅退出
4. **hash.c 中 record->key == NULL 警告**：key 是 char 数组非指针，比较地址永远非 NULL → 移除该条件

## VS Code 编译配置修复

**问题**：用户在 VS Code 中打开 main.c 点"运行"时，出现大量 `undefined reference` 链接错误（util_trim/hash_init/avl_insert 等全部未定义）。

**根因**：VS Code 的 C/C++ Code Runner 插件默认只编译当前打开的单个 .c 文件，不会自动链接其他源文件（avl.c/hash.c/util.c/record.c）。用户缺少 .vscode 配置目录。

**修复**：创建 `.vscode/tasks.json` 和 `.vscode/launch.json`：
- tasks.json：`gcc -Wall -Wextra -std=c99 -g src/*.c -I include -o campus_course.exe`，设为默认 build task
- launch.json：cppdbg 配置，preLaunchTask 调用上面的 build task，program 指向 campus_course.exe，externalConsole=true

**验证**：`gcc -Wall -Wextra -std=c99 -g src/*.c -I include -o campus_course.exe` 零错误零警告，生成 142KB 可执行文件。

**用户使用方式**：VS Code 中按 Ctrl+Shift+B 编译，按 F5 调试运行。
